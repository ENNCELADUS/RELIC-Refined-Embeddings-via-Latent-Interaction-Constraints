## 1. Core Methodology
(Theoretical foundation, mathematical formulations, and step-by-step algorithmic design)

- **Setting**: Unsupervised Domain Adaptation (UDA) without access to source data, using only a pre-trained source model. This preserves source data privacy and avoids heavy data transmission.
- **Core Idea - Source Hypothesis Transfer (SHOT)**: Fix the pre-trained source classifier module (the 'hypothesis' $h_s$) and learn a new domain-specific feature encoding module $g_t$ for the target domain. The goal is to optimize $g_t$ such that the target features align with the decision boundaries already established by $h_s$.
- **Information Maximization (IM) Loss**: Encourages individual target predictions to be confident (low entropy) while forcing the marginal class distribution across the entire target domain to be uniform (high entropy) to avoid catastrophic collapsing into a single class.
  - Entropy Loss: $L_{ent} = - \mathbb{E}_{x_t} \sum_{k=1}^K \delta_k(h_t(g_t(x_t))) \log \delta_k(h_t(g_t(x_t)))$
  - Diversity Loss: $L_{div} = \sum_{k=1}^K \hat{p}_k \log \hat{p}_k = D_{KL}(\hat{p}, \frac{1}{K}\mathbf{1}_K) - \log K$, where $\hat{p}$ is the mean network output embedding over the target batch.
- **Self-supervised Pseudo-labeling**: Because IM loss alone may align target features to the wrong class hypothesis, SHOT calculates $K$ target-domain centroids. These centroids are computed via a soft-weighted average of target features $\hat{g}_t(x_t)$ using the network's confidence scores. Unlabeled target instances are then assigned a pseudo-label by finding the nearest centroid via cosine distance.

## 2. Implementation Details
(Network architectures, loss functions, optimization strategies, and hyperparameter settings)

- **Source Model Architecture Modifications**:
  - The feature extractor $g_s$ consists of a backbone (e.g., ResNet-50) and a newly inserted **bottleneck layer** (`FC -> Batch Normalization (BN)`). The BN layer helps adjust for internal covariate shifts across domains.
  - The classifier $h_s$ is a dense layer augmented with **Weight Normalization (WN)**. Normalizing the weight vectors ensures the dot product acts identically to cosine similarity, creating tighter semantic clusters.
- **Source Model Pre-training Optimization**:
  - Trained using standard cross-entropy with **label smoothing** ($\alpha=0.1$) to encourage examples to lie in tight, evenly separated clusters, which improves the generalizability of the source hypothesis $h_s$.
- **Target Model (SHOT) Optimization**:
  - **Loss Function**: $\mathcal{L}(g_t) = L_{ent} + L_{div} - \beta \mathbb{E}_{x_t} \mathbf{1}[k=\hat{y}_t] \log \delta_k(h_t(g_t(x_t)))$
  - **Hyperparameters**: $\beta = 0.3$ for object recognition (0.1 for Digits).
  - **Optimizer**: Mini-batch SGD with momentum $0.9$, weight decay $1e-3$.
  - **Learning Rate**: Backbone uses a scheduled learning rate $\eta = \eta_0 \cdot (1 + 10 \cdot p)^{-0.75}$ with $\eta_0=0.01$, where $p \in [0, 1]$ is the training progress. The newly added bottleneck layers use a $10\times$ multiplier ($lr = 0.1$).
  - **Batch Size**: 64.
  - **Pseudo-labeling Updates**: Target centroids and pseudo-labels are updated epoch-by-epoch using the entire target dataset (or a large representative chunk), not per mini-batch.

## 3. Integration Requirements
(Input/output data formats, tensor dimensions, preprocessing steps, and pseudo-code)

- **Input Data Format**: Unlabeled target data samples stream, and a pre-trained source model partitioned into $g_s$ (encoder + bottleneck) and $h_s$ (classifier).
- **Preprocessing Steps**:
  1. Segregate the source model into $g_s$ and $h_s$.
  2. Initialize the target feature extractor $g_t \leftarrow g_s$.
  3. Freeze all parameters within the target classifier $h_t \leftarrow h_s$. 
- **Tensor Dimensions**: Target feature vector dimensions must exactly match the expected input size of the $h_s$ classifier bottleneck.
- **Pseudo-code Blueprint**:
```python
# Initialization
g_t.load_state_dict(g_s.state_dict())
h_t.load_state_dict(h_s.state_dict())
h_t.eval() # Freeze hypothesis

for epoch in range(num_epochs): # e.g., 15 epochs
    # Step 1: Self-supervised Pseudo-labeling (Epoch-level)
    g_t.eval()
    all_features, all_outputs = [], []
    with torch.no_grad():
        for x_t in target_dataloader:
            f_t = g_t(x_t)
            out_t = torch.softmax(h_t(f_t), dim=-1)
            all_features.append(f_t)
            all_outputs.append(out_t)
            
    all_features = torch.cat(all_features, dim=0)
    all_outputs = torch.cat(all_outputs, dim=0)
    
    # Compute centroids using confidence-weighted average
    centroids = compute_centroids(all_features, all_outputs)
    
    # Assign pseudo-labels by nearest centroid (Cosine Distance)
    pseudo_labels = get_nearest_centroid_labels(all_features, centroids)

    # Step 2: Mini-batch Feature Target Adaptation
    g_t.train()
    for batch_idx, (x_t, idx) in enumerate(target_dataloader): 
        # idx maps to the instance's pre-computed pseudo_label
        y_pseudo = pseudo_labels[idx] 
        
        f_t = g_t(x_t)
        logits = h_t(f_t)
        probs = torch.softmax(logits, dim=-1)
        
        # Calculate IM Loss
        entropy_loss = -torch.mean(torch.sum(probs * torch.log(probs + 1e-5), dim=-1))
        
        mean_probs = torch.mean(probs, dim=0)
        diversity_loss = torch.sum(mean_probs * torch.log(mean_probs + 1e-5))
        
        # Calculate Pseudo-label Loss
        ce_loss = cross_entropy(logits, y_pseudo)
        
        # Total Objective
        loss = entropy_loss + diversity_loss + beta * ce_loss
        
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        
        update_learning_rate_scheduler()
```
